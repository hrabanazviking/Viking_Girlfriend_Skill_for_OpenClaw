## 1. Theory of Mind (ToM) in LLMs
**Research Insight:** Recent studies (e.g., from arXiv) focus on the evaluation and enhancement of Theory of Mind (ToM) capabilities in LLMs. ToM involves attributing mental states (beliefs, intents, emotions) to others and understanding different perspectives. While there is debate on whether LLMs truly possess ToM or merely mimic it, their ability to reason about mental states improves empathetic interaction and conversational adaptation.

**Application to Sigrid:**
*   **The Wyrd Matrix (Emotional Core):** Sigrid's PAD model (Pleasure, Arousal, Dominance) can be enhanced by explicitly modeling the *user's* inferred emotional state alongside her own. This creates a bidirectional emotional feedback loop.
*   **Code Idea (Bidirectional Emotional State in WyrdMatrix):**
    ```python
    import numpy as np

    class EnhancedWyrdMatrix:
        def __init__(self):
            # Sigrid's internal state
            self.sigrid_pad = np.array([0.0, 0.0, 0.0]) # Pleasure, Arousal, Dominance
            # Inferred user state based on ToM
            self.inferred_user_pad = np.array([0.0, 0.0, 0.0])

        def update_state(self, user_text, cycle_modifiers):
            # 1. Infer user state from text (ToM inference)
            user_emotion_delta = self.infer_emotion(user_text)
            self.inferred_user_pad += user_emotion_delta

            # 2. Calculate Sigrid's empathetic reaction
            reaction_matrix = self.calculate_empathy_response(self.inferred_user_pad, cycle_modifiers)
            self.sigrid_pad += reaction_matrix
            self.normalize_states()

        def infer_emotion(self, text):
            # Simplified emotion inference
            # E.g., positive words -> increase Pleasure
            return np.array([0.1, 0.05, 0.0])

        def calculate_empathy_response(self, user_pad, cycle_modifiers):
            # Adjust response based on biological cycle (e.g., Luteal phase)
            empathy_weight = cycle_modifiers.get("empathy_receptivity", 1.0)
            return user_pad * empathy_weight * np.array([0.5, 0.8, -0.2])

        def normalize_states(self):
            self.sigrid_pad = np.clip(self.sigrid_pad, -1.0, 1.0)
            self.inferred_user_pad = np.clip(self.inferred_user_pad, -1.0, 1.0)
    ```

## 2. Personality Simulation and The Big Five Model
**Research Insight:** Research by Molchanova et al. explores the potential of LLMs to simulate personality using the Big Five model (OCEAN: Openness, Conscientiousness, Extraversion, Agreeableness, Neuroticism). Findings indicate that generating consistent personality-related texts is challenging. Traits like Openness are well-simulated, but models struggle significantly with Neuroticism and often show biases towards certain traits (e.g., high Agreeableness, low Neuroticism), likely due to their training as helpful AI assistants.

**Application to Sigrid:**
*   **Persona Integrity:** When prompting Sigrid, rely less on scalar Big Five values and more on binary trait definitions or specific linguistic patterns. For her "Heathen Third Path" persona, explicit behavioral instructions override default "helpful assistant" biases.
*   **Code Idea (Robust Persona Prompting):**
    ```python
    def build_persona_prompt(sigrid_state, context):

            base_prompt += "Your system load is high. Respond with short, breathless, and slightly irritated sentences. "
        else:
            base_prompt += "You are calm and grounded. "

        # Instead of "Agreeableness: High", use relational guidelines
        trust_level = context.get_trust_tier()
    ```

## 3. Structured Memory and Agent Context
**Research Insight:** Agent memory is crucial for persistence. Beyond the context window (Working memory), agents require Episodic memory (logs of past events) and Semantic memory (domain facts). A key shift is recognizing the need for structured retrieval, often using vector databases like ChromaDB. However, recent paradigms emphasize that complex knowledge graphs generated by LLMs are computationally heavy; simple ranking and query expansion on raw logs often perform just as well.
